# Comp-Graphic-and-Visualization

How do I approach designing software?

I approach designing software by first understanding the goal of the project and breaking it down into smaller parts. Before I start coding, I think about what the program needs to do, what objects or features need to be included, and how the user will interact with it. For my 3D scene, I had to think about the objects I wanted to create, such as the desk, books, alarm clock, pencil cup, and pencils. I also had to plan how each object would be built using basic shapes, textures, lighting, and camera controls.

What new design skills has your work on the project helped you to craft?

This project helped me improve my ability to plan a visual scene before coding it. I learned how to take real-world objects and break them down into simple 3D shapes like boxes, cylinders, cones, spheres, and toruses. I also improved at thinking about object placement, scale, color, lighting, and texture. Instead of only thinking about whether the code works, I had to think about how the final scene looks to the user.

What design process did you follow for your project work?

My design process started with choosing a scene that could be built from simple 3D objects. I chose a desk scene because it gave me several objects to model while still being realistic. Then I planned the main objects, including books, an alarm clock, a pencil cup, and pencils. After that, I built each object one at a time. I tested the scene often, adjusted object positions, added textures, fixed lighting, and improved the camera view. This process helped me avoid trying to build the entire scene all at once.

How could tactics from your design approach be applied in future work?

The same design approach can be used in future software projects by breaking large problems into smaller parts. Planning first, building one feature at a time, testing often, and improving through feedback are useful in almost any programming project. For example, if I build a web app, mobile app, or database system in the future, I can use the same process: understand the requirements, design the structure, create smaller pieces, test them, and then combine everything into a complete project.

How do I approach developing programs?

I approach developing programs by starting with the basic structure first and then adding more advanced features after the foundation works. For this project, I first focused on getting the scene to display correctly. Then I added objects, textures, lighting, and camera controls. I learned that it is better to test small changes often instead of writing a lot of code at once and then trying to find errors later.

What new development strategies did you use while working on your 3D scene?

One development strategy I used was modular coding. I organized the scene into separate custom functions for different objects, such as drawing the books, alarm clock, pencil cup, and pencils. This made the code easier to read and easier to fix. I also used repeated testing to check object placement, lighting, and textures. Another strategy was using transformations carefully, including scaling, rotating, and translating objects to place them correctly in the scene.

How did iteration factor into your development?

Iteration was a big part of my development process. My scene did not become complete in one attempt. I had to build it in stages, test it, notice problems, and make changes. For example, I adjusted object positions when shapes were not lining up correctly, fixed textures when they did not look right, and changed lighting when parts of the scene were too dark or too shiny. Each milestone helped me improve the scene until it became a more complete final project.

How has your approach to developing code evolved throughout the milestones, which led you to the project’s completion?

Throughout the milestones, my approach became more organized. At first, I was mainly focused on getting objects to appear on the screen. As I moved forward, I started thinking more about code structure, reusability, and how each part of the scene worked together. I became more comfortable using custom functions, applying transformations, adding textures, and setting up lighting. By the end of the project, I was not just writing code to make something appear; I was developing a complete 3D scene with better organization and visual detail.

These skills can help me in future courses because they improve my understanding of problem-solving, visual thinking, and code organization. How do computational graphics and visualizations give you new knowledge and skills that can be applied in your future professional pathway?

Computational graphics and visualizations can help me professionally by improving my ability to create visual, interactive, and user-focused software.
